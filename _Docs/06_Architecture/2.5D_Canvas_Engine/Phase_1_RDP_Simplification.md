# Phase 1: RDP Path Simplification

## Goal
To reduce the raw data points generated by the stylus or touch input before passing them into the `perfect_freehand` polygon engine. This prevents CPU bottlenecks and stuttering.

## Technical Context
When a user draws a line quickly, a 120Hz display logs hundreds of coordinates. Most of these points are redundant on straight lines. By passing these points through the Ramer-Douglas-Peucker (RDP) algorithm, we can strip away 60-70% of the useless data without changing the visual shape of the curve.

## Implementation Steps

1. **Create Utility File**
   - Create `lib/utils/rdp_simplifier.dart`.
   - Implement the recursive RDP algorithm. It will accept a `List<StrokePoint>` and a double `tolerance` (epsilon) parameter.

2. **Intercept Raw Input**
   - In `lib/controllers/canvas_controller.dart`, locate the `updateStroke` and `endStroke` methods.
   - During `updateStroke`, keep all raw points so the live line remains perfectly responsive to the user's micro-movements.
   
3. **Apply Simplification on Release**
   - In `endStroke()`, right before saving the finalized `Stroke` to the `layersNotifier` state, pass the raw points through the `rdp_simplifier` with a tuned tolerance (e.g., `0.5` or `1.0`).
   - The finalized `Stroke` object is stored with the significantly smaller list of points.

## Success Criteria
- [ ] A fast stroke generates 70% fewer data points in memory.
- [ ] The visual shape of the finalized stroke remains identical to the live stroke.
- [ ] No visible stutter when the pen is lifted from the screen.
